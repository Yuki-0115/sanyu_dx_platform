案件管理

案件の登録から完了までの一元管理

案件管理とは

案件管理は、工事案件の全ライフサイクルを管理する機能です。引合いから入金完了まで、すべてのデータは案件に紐づいて管理されます。

引合い
→
案件登録
→
見積作成
→
受注
→
施工
→
完工
→
請求・入金

案件IDがすべてのキー

見積、日報、請求、入金など、すべてのデータは案件IDで紐づけられます。これにより、案件単位での原価管理・収支管理が可能になります。

案件の登録

登録項目

項目名 必須 説明
案件コード - 自動採番（PJ + 年月 + 連番）
案件名 ◯ 工事名称
顧客 - 顧客マスタから選択
案件種別 ◯ 通常案件 / その他（小工事・常用）
受注フロー ◯ 通常 / 口頭先行
現場住所 - 工事現場の住所
営業担当 - 営業担当社員
工務担当 - 工務担当社員
施工担当 - 施工管理担当社員
予定開始日 - 工事予定開始日
予定終了日 - 工事予定完了日

登録手順

  1. メニューから「案件一覧」を選択 案件一覧画面が表示されます。
  2. 「新規案件」ボタンをクリック 案件登録フォームが表示されます。
  3. 基本情報を入力 案件名、顧客、担当者などを入力します。
  4. 「登録」ボタンで保存 案件コードが自動採番され、案件が作成されます。

案件種別

通常案件（regular）

標準的な工事案件。見積作成→4点チェック→実行予算→日報→請求の完全なフローを経由します。

  • 見積作成が必須
  • 4点チェック完了で受注確定
  • 実行予算の作成が必要

その他（misc）

小規模工事や常用作業など、簡略化されたフローで管理する案件。

  • 見積は省略可能
  • 4点チェックは簡易版
  • 即時着工可能

受注フロー

通常フロー（standard）

見積提出
→
顧客承認
→
注文書受領
→
4点チェック
→
受注確定

見積承認後、正式な注文書を受領してから受注確定とするフローです。

口頭先行フロー（oral_first）

口頭受注
→
着工
→
注文書受領（後日）

口頭で受注を受け、先に着工し、後から正式な注文書を受領するフローです。

口頭先行フローの注意点

口頭受注から14日を超えて注文書が未受領の場合、アラートが表示されます。リスク管理のため、早めの注文書取得を推奨します。

ステータス管理

ステータス 説明 できること 次のステップ
draft 下書き 案件情報の編集、見積作成 見積提出
estimating 見積中 見積の編集・提出 4点チェック完了
ordered 受注済 実行予算作成 着工準備
preparing 着工準備中 5点ゲート確認 着工
in_progress 施工中 日報入力、原価管理 完工
completed 完工 請求書発行 請求
invoiced 請求済 入金確認 入金完了
paid 入金済 案件クローズ クローズ
closed クローズ 閲覧のみ -

案件詳細画面

案件詳細画面では、案件に関連するすべての情報にアクセスできます。

基本情報タブ

案件の基本情報、担当者、ステータスを表示・編集します。

見積タブ

見積書の一覧、作成、PDF出力を行います。

4点チェックタブ

受注確定のための4点チェックを管理します。

実行予算タブ

実行予算の作成・確認を行います。

日報タブ

日報の一覧、入力を行います。

現場台帳タブ

原価集計、予算対比を確認します。

請求タブ

請求書の作成・発行を行います。

入金タブ

入金の記録・確認を行います。

書類タブ

関連書類のアップロード・管理を行います。

Google Drive連携

案件が登録されると、自動的にGoogle Driveにフォルダが作成されます。

フォルダ構成

案件フォルダ (PJ202601001_新宿ビル外壁改修工事) ├── 01_見積 ├── 02_契約・注文書 ├── 03_施工図・図面 ├── 04_安全書類 ├── 05_日報 ├── 06_請求書 ├── 07_完了報告 └── 99_その他
自動同期

システムから出力したPDF（見積書、請求書など）は自動的に対応するフォルダにアップロードされます。
