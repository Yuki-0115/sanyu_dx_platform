マスタ管理

顧客・協力会社・社員の基本データ管理

マスタ管理とは

マスタデータは、システム全体で共通して使用される基本データです。案件登録や日報入力の際に参照されるため、事前に正確なデータを登録しておくことが重要です。

マスタデータの種類

  • 顧客マスタ - 発注元の顧客情報
  • 協力会社マスタ - 外注先・協力会社情報
  • 社員マスタ - 自社社員の情報

顧客マスタ

発注元となる顧客の情報を管理します。案件登録時に顧客を選択するため、事前登録が必要です。

登録項目

項目名 必須 入力形式 説明
顧客コード - 自動採番 CL + 連番で自動生成（例：CL001）
顧客名 ◯ テキスト 正式な会社名または個人名
顧客名（カナ） - カタカナ 検索・並び替え用のフリガナ
担当者名 - テキスト 主要連絡先となる担当者名
電話番号 - 電話番号 ハイフン区切り（例：03-1234-5678）
FAX番号 - 電話番号 FAX番号
メールアドレス - メール 連絡先メールアドレス
郵便番号 - 郵便番号 7桁（例：123-4567）
住所 - テキスト 請求書送付先等の住所
与信限度額 - 金額 取引上限金額（円）
支払条件 - 選択 月末締め翌月払い等
備考 - テキスト 特記事項など

登録手順

  1. メニューから「マスタ管理」→「顧客」を選択 顧客一覧画面が表示されます。
  2. 「新規顧客」ボタンをクリック 顧客登録フォームが表示されます。
  3. 顧客情報を入力 必須項目（顧客名）は必ず入力してください。
  4. 「登録」ボタンで保存 顧客コードが自動採番され、登録完了です。
与信限度額について

与信限度額を設定すると、その顧客への未入金請求額が限度額を超えた場合にアラートが表示されます。新規取引先は必ず設定することを推奨します。

協力会社マスタ

外注先・協力会社の情報を管理します。日報での外注入力や仮社員相殺で使用します。

登録項目

項目名 必須 入力形式 説明
会社コード - 自動採番 PT + 連番で自動生成
会社名 ◯ テキスト 協力会社の正式名称
代表者名 - テキスト 代表者または担当者名
電話番号 - 電話番号 連絡先電話番号
メールアドレス - メール 連絡先メール
仮社員有無 - チェック 仮社員がいる場合はチェック
相殺ルール - 選択 相殺計算のルール
標準人工単価 - 金額 日報入力時のデフォルト単価
銀行口座 - テキスト 振込先口座情報

仮社員がいる場合の設定

協力会社から派遣された仮社員がいる場合、相殺処理のための追加設定が必要です。

相殺ルール

  • 標準 - 給与＋社保を支払い、人工売上と相殺
  • 固定額 - 毎月固定額での相殺
  • なし - 相殺なし（通常の外注扱い）

社会保険率

給与に対する社会保険料の負担率を設定します。デフォルトは15%です。

社員マスタ

自社社員の情報を管理します。日報の出面入力、案件担当者の設定、システムログインに使用します。

登録項目

項目名 必須 入力形式 説明
社員番号 - 自動採番 EMP + 連番で自動生成
氏名 ◯ テキスト 社員の氏名
氏名（カナ） - カタカナ フリガナ
雇用形態 ◯ 選択 正社員 / 臨時社員
役職 - 選択 職長 / 作業員 / 事務 等
部署 - 選択 所属部署
入社日 - 日付 入社年月日
メールアドレス ◯* メール ログイン用（システム利用者のみ）
権限 ◯* 選択 システム権限（システム利用者のみ）

* システムにログインする社員のみ必須

雇用形態と原価計算

雇用形態 原価計上 説明
正社員 労務費として計上 人工数 × 正社員単価で原価計算
臨時社員 労務費として計上 人工数 × 臨時社員単価で原価計算

権限について

権限 閲覧 編集 説明
admin 全機能 全機能 システム管理者。すべての機能にアクセス可能
management 全機能 制限あり 経営層。ダッシュボード閲覧、承認機能
accounting 経理関連 経理関連 経理担当。請求・入金・経費処理
sales 営業関連 営業関連 営業担当。案件・見積作成
engineering 工務関連 工務関連 工務担当。実行予算・原価管理
construction 施工関連 日報入力 施工担当。日報入力・現場管理
worker 自分の情報 - 作業員。閲覧のみ（Worker Web用）
権限設定の注意

権限は慎重に設定してください。特に「admin」権限は必要最小限の人数に留めることを推奨します。

データ管理のポイント

データの正確性

  • 会社名・氏名は正式名称で登録する
  • 連絡先は最新の情報を維持する
  • 退職者・取引終了先は「無効」にして履歴を残す

重複登録の防止

  • 登録前に既存データを検索して確認する
  • 略称や表記揺れに注意（例：(株) と 株式会社）
  • 重複が見つかった場合は統合処理を行う

定期的なメンテナンス

  • 連絡先の変更があれば速やかに更新
  • 年1回は全データの棚卸しを推奨
  • 使用していないデータは「無効」に設定
